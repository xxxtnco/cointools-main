# Bybit Trading Assistant

一个功能强大的 Bybit 交易所浏览器扩展助手，支持智能网格交易和自动交易功能。

## 功能特性

### 1. 智能网格交易系统
- 动态价格跟踪 - 网格中心随当前价格移动
- 响应式网格重置 - 价格超出阈值时自动调整
- 利润率控制 - 通过固定买卖价差百分比实现盈利
- 完善资金管理 - 实时追踪资金使用情况
- 多层止损保护 - 支持单格止损和整体止损

### 2. 速刷交易量模式
- 高频自动交易 - 快速完成交易量任务
- Maker 模式 - 低买高卖策略，降低手续费成本
- 自适应价格调整 - 根据市场情况智能调整
- 实时统计监控 - 交易量、手续费、盈亏一目了然

## 安装配置

### 前置要求
- Chrome 浏览器（或基于 Chromium 的浏览器）
- Bybit 交易所账户
- Bybit API 密钥（Key 和 Secret）

### 安装步骤

1. **下载项目**
   ```bash
   git clone https://github.com/xxxtnco/cointools-main.git
   cd cointools-main
   ```

2. **加载扩展**
   - 打开 Chrome 浏览器
   - 访问 `chrome://extensions/`
   - 开启右上角的"开发者模式"
   - 点击"加载已解压的扩展程序"
   - 选择项目文件夹

3. **获取 API 密钥**
   - 登录 [Bybit 官网](https://www.bybit.com)
   - 进入 API 管理页面
   - 创建新的 API Key
   - 权限设置：需要开启"现货交易"权限
   - 保存 API Key 和 Secret（Secret 只显示一次，请妥善保存）

4. **配置扩展**
   - 访问 Bybit 现货交易页面
   - 点击浏览器工具栏的扩展图标打开侧边栏
   - 在"API 配置"区域输入：
     - API Key
     - API Secret
   - 点击"保存配置"

## 使用教程

### 网格交易模式

**适用场景**：横盘震荡行情，通过频繁买卖赚取差价

1. **基础配置**
   ```
   交易对：NXPCUSDT（或其他币对）
   投入资金：100 USDT（建议从小额开始）
   网格数量：10（可根据需要调整 5-20）
   利润百分比：2.5%（每次交易的目标利润）
   ```

2. **高级配置**
   ```
   资金使用比例：100%（可设置 50-100%）
   重置阈值：15%（价格偏离超过此值自动重置网格）
   单格止损：5%（单个网格亏损超过 5% 止损）
   整体止损：10 USDT（总亏损达到 10 USDT 停止）
   检查间隔：10 秒
   ```

3. **启动步骤**
   - 填写配置参数
   - 点击"启动网格交易"
   - 系统自动：
     - 获取当前价格
     - 计算网格价位
     - 布置买卖订单
     - 实时监控成交
     - 自动重新挂单

4. **监控指标**
   - 当前价格 vs 网格中心
   - 活跃订单数量（买单/卖单）
   - 成交统计（成交次数、总金额）
   - 资金使用情况
   - 止损触发次数

### 速刷交易量模式

**适用场景**：需要快速完成交易量任务

1. **基础配置**
   ```
   交易对：NXPCUSDT
   刷单间隔：6 秒（建议 3-10 秒）
   资金使用：100%
   价格偏移：0.08%（越小越容易成交，但手续费损失越大）
   ```

2. **高级配置**
   ```
   Maker 模式：开启（低买高卖，减少手续费）
   订单超时：45 秒（超时未成交则取消）
   自适应模式：开启（超时后自动缩小价格偏移）
   最大亏损：50 USDT（累计亏损达到此值停止）
   ```

3. **启动步骤**
   - 填写刷单配置
   - 点击"开始刷交易量"
   - 系统循环执行：
     - 查询余额
     - 下买单 → 等待成交
     - 下卖单 → 等待成交
     - 统计手续费和盈亏

4. **实时统计**
   - 运行时长
   - 刷单次数
   - 累计交易量（USDT）
   - 手续费支出
   - 净亏损金额

## 重要注意事项

### ⚠️ 风险警示

1. **交易风险**
   - 加密货币交易存在高风险，可能导致本金损失
   - 网格交易在单边行情中可能产生较大亏损
   - 速刷模式会产生手续费成本
   - 请使用闲置资金，不要投入超过承受能力的资金

2. **技术风险**
   - 网络延迟可能导致订单执行失败
   - API 限频可能导致交易中断
   - 浏览器关闭或网络断开会停止运行
   - 建议保持浏览器开启和网络稳定

3. **合规性**
   - 请遵守当地法律法规和交易所规则
   - 不要用于市场操纵或违规刷量
   - 仅用于个人合法交易需求
   - 交易所可能对异常交易行为进行限制

### 💡 使用建议

1. **资金管理**
   - 建议从小额资金开始测试（如 10-50 USDT）
   - 设置合理的止损限额
   - 不要投入全部资金到单一交易对

2. **参数优化**
   - 网格数量：震荡幅度大选 5-10 格，小选 15-20 格
   - 利润百分比：建议 1-3%，太高不易成交
   - 价格偏移：Maker 模式建议 0.05-0.1%

3. **监控维护**
   - 定期查看交易记录和盈亏情况
   - 行情剧烈波动时建议暂停交易
   - 及时更新止损参数

4. **安全建议**
   - API Key 权限设置为仅交易，不开启提现
   - 定期更换 API 密钥
   - 不要在公共场合使用或泄露 API 信息
   - 建议设置 IP 白名单限制

### 🔧 故障排查

1. **无法连接**
   - 检查网络连接是否正常
   - 确认 API Key 和 Secret 正确
   - 检查 API 权限是否开启现货交易

2. **订单失败**
   - 检查余额是否充足
   - 确认交易对是否正确
   - 查看是否触及 API 限频

3. **价格不更新**
   - 刷新页面重新连接 WebSocket
   - 检查浏览器控制台是否有错误

## 技术架构

- **订单系统**：Cookie 模拟浏览器下单
- **查询系统**：Bybit V5 API 查询订单和余额
- **价格监控**：双 WebSocket 实时监控（价格流 + 订单流）
- **资金管理**：FundManager 追踪资金使用
- **网格管理**：GridLevelManager 管理每层网格状态
